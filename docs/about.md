# About

## Background

I'm a technical writer and technical lead with over 20 years of professional experience spanning API documentation, content strategy, and software delivery.

Since 2007, I've been the sole author and maintainer of the complete developer documentation for a major international travel technology platform. The documentation suite covers **320+ articles across 6 API products**, including:

- SOAP/XML API reference documentation with element-by-element schema breakdowns
- Developer guides for 30+ API workflows (shopping, booking, payment, exchanges, cancellations, refunds)
- Carrier-specific implementation and accreditation guides for 7+ international rail operators
- Biweekly release notes with XSD schema diff analysis over 5+ years
- Conceptual overviews, FAQs, and how-to summaries

## Docs-as-code approach

I designed and implemented the entire documentation publishing pipeline:

- **Source format**: Git-managed Markdown with YAML frontmatter
- **Version control**: GitHub and GitLab repositories
- **CI/CD**: GitHub Actions and GitLab CI pipelines that auto-deploy changed articles on merge to main
- **CMS integration**: Bidirectional sync with Document360 via a custom Python CLI tool I designed (built with AI-assisted coding)
- **Content tooling**: Migration scripts, schema diff parsers, HTML-to-Markdown converters, and bulk content transformation utilities — all specified and directed by me, developed with the assistance of AI coding agents
